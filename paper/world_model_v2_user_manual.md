# GIS Data Agent — 世界模型 v2 操作手册

版本：v2.0.0-alpha  
适用范围：璧山区耕地布局优化  
更新日期：2026-05-01

---

## 一、功能简介

世界模型 v2 是 GIS Data Agent 中的一个独立功能模块，基于 paper9（Dual-Layer Geospatial Dreamer / Targeted DAgger）的研究成果。

它的作用是：

> 对璧山区全县 2,600 个空间地块、52,515 个宗地的耕地布局进行优化推演，输出一个"建议调整方案"，包括哪些地块应从耕地转为林地、哪些应从林地转为耕地，以降低耕地平均坡度、提升空间连片度。

当前版本特点：
- 使用璧山区内置数据，无需上传文件
- 使用离线训练好的最佳策略 checkpoint
- 在线只做推理，不做重新训练
- 结果自动推送到地图面板

## 三、操作步骤

### Step 1：进入"世界模型v2"面板

登录后，在右侧工作台（DataPanel）中：

1. 点击顶部的 **"智能分析"** 分组
2. 在 tab 列表中找到 **"世界模型v2"**
3. 点击进入

### Step 2：查看模型状态

进入面板后，顶部会显示当前模型状态：

| 状态 | 含义 |
|------|------|
| **模型就绪**（绿色） | checkpoint 存在，可以运行 |
| **模型不可用**（黄色） | checkpoint 缺失或加载失败 |
| **检测中...**（灰色） | 正在检查模型状态 |

状态旁边会显示版本号和支持区域。

### Step 3：确认提示信息

面板中会有一行提示：

> "当前仅支持璧山区内置数据，无需上传文件"

这说明你不需要做任何数据准备，直接运行即可。

### Step 4：设置评估回合数

在"评估回合数"输入框中填写数字。

| 场景 | 建议值 | 说明 |
|------|--------|------|
| 快速演示 | 3 | 约 1 分钟出结果 |
| 标准使用 | 10 | 约 3 分钟，结果更稳定 |
| 严格验证 | 20-30 | 约 5-10 分钟，适合正式汇报 |

评估回合数的含义：系统会用当前最佳策略跑 N 次完整优化 episode，然后选出 reward 最高的那次作为最终输出。回合数越多，选出的结果越稳定。

### Step 5：点击"运行耕地布局优化"

点击蓝色按钮后，按钮会变成"优化运行中..."状态。

此时系统内部正在：
1. 加载璧山区全县环境数据（首次约 45-60 秒）
2. 加载训练好的策略 checkpoint
3. 运行 N 个 evaluation episode
4. 选出最佳 episode
5. 生成优化后布局和变化差异的 GeoJSON 图层
6. 推送地图更新

### Step 6：查看结果

运行完成后，面板中会出现结果表格。

## 四、结果字段解读

### 4.1 数值指标

| 字段 | 含义 | 怎么看 |
|------|------|--------|
| **总奖励** | 综合优化得分 | 正值 = 优化有效；负值 = 优化效果不佳 |
| **坡度变化** | 耕地平均坡度的变化百分比 | 负值 = 坡度降低（好）；正值 = 坡度升高（差） |
| **连片度变化** | 耕地空间连片度的变化 | 正值 = 更集中（好）；负值 = 更分散（差） |
| **变更地块数** | 发生土地利用类型变化的地块总数 | 越多说明调整幅度越大 |
| **耕地→林地** | 从耕地转为林地的地块数 | 通常是高坡度耕地被退出 |
| **林地→耕地** | 从林地转为耕地的地块数 | 通常是低坡度林地被补入 |
| **最佳回合** | 在 N 个 episode 中，第几个是最优的 | 仅供参考 |
| **耗时** | 本次运行总耗时（秒） | 首次运行会较慢（需加载环境） |

### 4.2 什么样的结果算"好"

| 指标 | 好的范围 | 说明 |
|------|----------|------|
| 总奖励 | > 0 | 正值说明优化方向正确 |
| 坡度变化 | < -0.05% | 负值越大说明坡度降低越多 |
| 连片度变化 | > 0 | 正值说明空间更集中 |
| 变更地块数 | 100-400 | 太少说明没怎么调整；太多可能过于激进 |

### 4.3 什么样的结果说明模型不稳定

| 现象 | 可能原因 |
|------|----------|
| 总奖励为较大负值（< -50） | 当前 checkpoint 在这次初始状态下表现不佳 |
| 坡度变化为正值 | 优化方向跑偏，策略可能在追 reward 而非真正降坡度 |
| 变更地块数为 0 | 策略没有学到有效动作 |
| 连片度大幅下降 | 策略过度追求坡度而牺牲了空间连片性 |

遇到这些情况时，可以：
- 增加评估回合数重跑一次
- 或者等后续更好的 checkpoint 替换后再试

## 五、地图图层说明

运行成功后，地图面板会自动加载两个新图层。

### 5.1 图层 A：优化后布局

| 属性 | 值 |
|------|-----|
| 图层名称 | 优化后布局 (Optimized Layout) |
| 图层类型 | 分类着色 (categorized) |
| 分类字段 | Opt_Type |

| 颜色 | 含义 |
|------|------|
| 灰色 (#D3D3D3) | 其他用地 |
| 金色 (#FFD700) | 耕地 |
| 深绿 (#228B22) | 林地 |

这个图层展示的是：优化完成后，每个地块的最终土地利用类型。

### 5.2 图层 B：变化差异

| 属性 | 值 |
|------|-----|
| 图层名称 | 变化差异 (Change Diff) |
| 图层类型 | 分类着色 (categorized) |
| 分类字段 | Change_Type |

| 颜色 | 含义 |
|------|------|
| 红橙色 (#FF4500) | 耕地 → 林地（退耕还林） |
| 蓝色 (#1E90FF) | 林地 → 耕地（补充耕地） |

这个图层只显示发生了变化的地块，没有变化的地块不显示。

### 5.3 推荐查看顺序

1. 先单独查看 **优化后布局**，了解整体格局
2. 再叠加 **变化差异**，看具体哪些地方被调整
3. 放大到具体乡镇，观察局部调整是否合理
4. 结合数值指标判断整体方案质量

### 5.4 地图中心和缩放

运行后地图会自动定位到：
- 中心：璧山区（约 29.59°N, 106.22°E）
- 缩放级别：12

你可以手动缩放和平移来查看不同区域的细节。

## 八、常见问题

### Q1：运行后地图没有更新？
- 检查浏览器控制台是否有错误
- 尝试手动刷新页面
- 确认后端服务正在运行

### Q2：结果每次都不一样？
- 这是正常的。每次运行的初始状态有随机性
- 增加评估回合数可以让结果更稳定
- 多次运行取最佳结果是推荐做法

### Q3：总奖励为负值，是不是失败了？
- 不一定。轻微负值（-10 以内）仍然可能有有效的坡度改善
- 重点看坡度变化和连片度变化
- 如果总奖励低于 -50，建议重跑

### Q4：能不能换成别的区县？
- 当前版本不支持
- 后续版本计划支持用户上传新区县数据
- 目前仅限璧山区内置数据

### Q5：运行时间太长？
- 首次运行需要加载环境数据（约 45-60 秒），后续会快一些
- 评估回合数越多，耗时越长
- 建议日常使用 3-10 个回合

## 九、技术参数（供参考）

| 参数 | 值 |
|------|-----|
| 模型版本 | 2.0.0 |
| 训练方法 | Targeted DAgger (threshold-based) |
| Checkpoint | dagger_ensemble_seed3_iter2_best.zip |
| 环境规模 | 13 乡镇, 2,600 地块, 52,515 宗地 |
| 观测维度 | 44,212 |
| 动作空间 | 2,600 (block selection) |
| 每 episode 步数 | 100 |
| 每步交换数 | 最多 5 对 |
| 策略架构 | ParcelScoringPolicy (MaskablePPO) |
| 运行硬件 | CPU (无需 GPU) |

## 十、版本说明

### 当前版本：v2.0.0-alpha
- 璧山区单点部署
- 固定 checkpoint
- 在线推理，不在线训练
- 适合原型验证和演示

- v2.0.0-beta：替换为更优 checkpoint（Top-K 或更多 seeds 训练后）

- v3.0.0：支持在线 dream rollout 和情景推演
